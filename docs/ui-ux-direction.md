# UI/UX Direction

## Design Philosophy

GeoWorkbench should feel:

- modern
- industrial
- data-centric
- clean
- AI-native
- geology-first

The interface should avoid looking like legacy mining software.

---

# Core UX Principles

## 1. Visualization First

Geologists think visually.

The product should prioritize:

- borehole tracks
- lithology columns
- seam continuity
- depth relationships
- geological sections
- geophysical curves

---

## 2. AI as Assistant, Not Controller

AI suggestions should appear as:

- overlays
- recommendations
- side panels
- confidence indicators

Geologists remain in control.

---

## 3. Low Friction Editing

Editing geological intervals should feel fast.

Features:

- drag intervals
- split intervals
- merge intervals
- quick lithology tagging
- inline notes

---

## 4. Multi-Panel Workspace

Recommended layout:

- left navigation
- central geological workspace
- right AI assistant/context panel
- bottom optional data/log panel

---

# Desktop + Web Strategy

## Desktop

Primary initial deployment.

Reasons:

- mining environments
- local datasets
- offline usage
- simpler enterprise deployment

Recommended:

- Tauri

---

## Web

Secondary expansion path.

Useful for:

- collaboration
- cloud sync
- remote teams
- enterprise dashboards
