# CTSJ-30 Experiment Strategy

The CTSJ workbook looks like a useful corrected or near-corrected descriptive lithology sheet. For product demos, we should not manually damage the original file. Instead, we keep the source workbook intact and generate controlled experiment fixtures.

## Fixture Set

Generated by:

```powershell
python scripts\build_ctsj_experiment.py
```

Outputs:

| File | Purpose |
| --- | --- |
| `sample-data/ctsj-30/ctsj-30-clean.json` | Normalized clean extract from the workbook. |
| `sample-data/ctsj-30/ctsj-30-ai-test.json` | Same borehole with known planted issues. |
| `sample-data/ctsj-30/ctsj-30-experiment-manifest.json` | Ground truth list of planted issues. |
| `sample-data/ctsj-30/ctsj-30-synthetic-curves.csv` | Synthetic gamma, density, and resistivity curves aligned to lithology. |

## Planted Issues

The AI test fixture introduces deterministic issues:

- missing lithology code
- interval overlap
- interval gap
- recovery greater than thickness
- RQD percent greater than 100
- invalid depth range
- curve/lithology mismatch zones

These are intentionally simple. They are not meant to pretend that AI has solved geological interpretation. They let us demonstrate the assistance loop:

1. Import clean/corrupted workbook-derived data.
2. Run deterministic validation checks.
3. Let the assistant summarize issues and suggest likely corrections.
4. Show the geologist source row, depth, affected field, and evidence.
5. Let the geologist accept, edit, or reject each suggestion.

## Synthetic Curves

Synthetic curves are generated from lithology classes:

- coal: lower gamma, lower density, higher resistivity
- shale/carbonaceous shale: higher gamma, higher density, lower resistivity
- sandstone: medium gamma, higher density, moderate resistivity

Two known curve anomalies are planted:

- 82m to 86m: coal-like response inside a sandstone interval
- 334m to 338m: shale-like response inside a coal interval

This lets us test curve/lithology inconsistency detection before real LAS/geophysical logs arrive.

## Why This Helps

This gives us a repeatable product conversation without depending on perfect customer data availability:

- We can show how the system protects geologists from common spreadsheet mistakes.
- We can show source-linked correction suggestions.
- We can test validation and assistant workflow end to end.
- We can later swap synthetic curves with actual LAS/CSV curves using the same canonical model.

The important discipline is that every planted issue must be listed in the manifest. That gives us measurable acceptance criteria: the system should find the expected issues and avoid noisy false positives.
